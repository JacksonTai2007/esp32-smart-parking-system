# 外观制作 + 2 天交视频速成指南

面向「时间紧、要交视频给老师、手头只有 A4 纸 / 彩色卡纸 / 胶带」的场景。
目标：**2 天内交出一段 ~2 分钟、能看出完整功能的演示视频**，外观用纸快速搭。

> 核心思路：**录视频不必和红外传感器死磕**。固件已内置「演示 / 模拟模式」
> （`config/Settings.h` 的 `ENABLE_SIM_MODE`），开启后车位状态由网页按钮
> （车辆驶入 / 驶离）驱动，整条链路（识别进出 → 计时 → 计费 → 蜂鸣 → OLED → 网页）
> 与真机完全一致。你只要能**烧录 ESP32 + 手机连热点**，就能录出完整闭环。

---

## 一、最小硬件（录视频只需三件套）

| 件 | 作用 | 必需性 |
| --- | --- | --- |
| ESP32 开发板 + USB 线 | 主控 + 供电 + 烧录 | 必需 |
| OLED SSD1306（I2C，GPIO 21/22） | 屏上显示车位地图 / 收入，镜头里很出片 | 强烈建议 |
| 有源蜂鸣器（GPIO 14） | 入场 1 响 / 离场 2 响 / 满位长响，视频有声更真实 | 建议 |
| SG90 舵机 + 卡纸闸杆（GPIO 13） | 出入口道闸，镜头里"抬杆 / 落杆"很出片；不接也能在网页看状态 | 建议 |
| 红外避障传感器 ×2~4 | **模拟模式下可不接**；想录真实镜头才接 | 可选 |
| 小车模型 ×4 | 摆进纸车位，镜头里就是"真车" | 建议 |

接线权威定义见 [`config/Pins.h`](../firmware/parking-system/config/Pins.h) 与
[docs/pinout.md](pinout.md)；OLED 与蜂鸣器即使不接也能编译运行（对应 `ENABLE_*` 开关）。

## 二、开启模拟模式（一行改动）

编辑 [`firmware/parking-system/config/Settings.h`](../firmware/parking-system/config/Settings.h)：

```cpp
#define ENABLE_SIM_MODE      1   // 录视频 / 无传感器演示时设 1；真机演示时改回 0
```

烧录后：

1. 手机连热点 **`ESP32-Parking`**（密码 `parking123`），浏览器开 `http://192.168.4.1/`
   （或接了路由器时用串口打印的 IP）。
2. 网页标题旁出现「模拟模式」标签，底部出现 **演示控制** 按钮：
   `P1 🚗 驶入` / `P1 🚙 驶离` …
3. 点按钮即可模拟车辆进出，OLED、蜂鸣器、收入统计同步联动。

> 真机演示（接了红外传感器）时把 `ENABLE_SIM_MODE` 改回 `0` 重新烧录即可，
> 同一套业务逻辑不变，只是输入从"按钮"换回"传感器"。

## 三、A4 / 卡纸快速搭停车场模型（约 2~3 小时）

成品是一块"停车场俯视沙盘"，和网页的俯视平面图相互呼应，镜头一扫就懂。

### 材料
- 深灰 / 黑色卡纸 1~2 张（当沥青地面）；没有就用 A4 平铺후涂色
- 白色或浅色卡纸（裁成细条做车位线）、黄色卡纸（做入口箭头 / 道路中线）
- 透明胶 / 双面胶、剪刀 / 美工刀、尺、笔
- 小车模型 ×4

### 步骤
1. **底板**：把 2~4 张 A4 用胶带在背面拼成 A3/A2 大小，正面铺深灰卡纸当地面。
2. **画车位**：用白色卡纸细条（或白胶带）贴出 **4 个停车位**，建议每位
   约 **90mm 宽 × 130mm 长**（放得下一辆小车）；在每位角上写 **P1~P4**。
3. **车道与出入口**：在车位前方留一条 ~60mm 宽的"车道"，中间用黄色卡纸虚线点出
   中心线，端头贴箭头并写「入口 / 出口」——和网页底部的 `⟵ 入口 · 出口 ⟶` 一致。
4. **收费 / 管理室**：用卡纸折一个小盒子放在角落，把 **ESP32 + 面包板藏进去**，
   正面挖一个窗口露出 **OLED**，侧面戳小孔让蜂鸣器声音透出来。
5. **招牌**：一张卡纸写 **「智能停车场 Smart Parking」**，折个底座立在入口。
6. **出入口道闸**：把一根卡纸条（闸杆）粘在 **SG90 舵机摆臂**上，舵机用双面胶/热熔胶
   固定在入口一侧；信号线（橙）接 GPIO 13，红/棕接 5V/GND。落杆时闸杆水平挡住车道、
   抬杆时竖起放行——车进/出会**自动抬杆、几秒后自动落杆**，镜头很出彩。方向反了就在
   `Settings.h` 把 `GATE_SERVO_OPEN_DEG` / `GATE_SERVO_CLOSED_DEG` 对调。
   没有舵机也行：闸杆用图钉别住手动掰，配合网页「道闸」状态一起演示。
7. **摆车**：把小车模型放进各车位——录入场镜头时配合网页按钮一起放/拿。

> 接真实红外时：把红外模块藏在每个车位**车头位置的下方或侧立**，杜邦线从底板
> 背面走线到管理室的 ESP32（GPIO 32/33/34/35）；模拟模式录视频则可省略，纯摆车即可。

## 四、录视频脚本（目标 ~2 分钟）

建议：**电脑浏览器开 `http://192.168.4.1` 录屏**（画面最清晰，电脑也连 ESP32 热点），
**另一台手机拍纸模型 + 小车**，后期画中画混剪；安静环境收蜂鸣声。

| 时间 | 画面 | 操作 / 旁白 |
| --- | --- | --- |
| 0:00–0:10 | 扫过纸停车场 + 招牌 | 一句："基于 ESP32 的智能停车场管理系统" |
| 0:10–0:25 | OLED + 网页初始页 | 上电启动，4 个车位全空、剩余 4/4、收入 0 |
| 0:25–0:55 | 点「P1 驶入」+ 放小车进 P1 | **道闸自动抬杆**（SG90 转动）放行 + 蜂鸣 1 响；网页 P1 变 🚗 开始实时计时，剩余 −1，OLED 同步；几秒后道闸自动落杆 |
| 0:55–1:15 | 再点 P2、P3 驶入 | 剩余车位持续减少，多车占位（每次进场道闸都抬杆） |
| 1:15–1:30 | 点最后一个车位驶入 | **满位横幅**变红 + 蜂鸣长响 |
| 1:30–2:00 | 点「P1 驶离」+ 拿走小车 | **道闸抬杆放行** + 蜂鸣 2 响；弹出本次费用，累计收入 / 次数更新，最近记录追加一条，数秒后落杆 |
| 2:00–2:10 | 点网页「开闸 / 落闸」 | 演示远程手动控制道闸，SG90 实时抬杆 / 落杆 |
| 2:10–2:25 | 点「清零累计收入」 | 回到初始；一句总结：非阻塞 millis 调度 + 模块化 + 整数分计费 |

讲解可参考 [docs/demo-script.md](demo-script.md) 的"答辩讲解要点"。

## 五、2 天时间表

**第 0 步（准备，约 1 小时，越早越好）**
- 拉最新分支；`./scripts/compile.sh --install-deps` 装环境并编译
- 接最小三件套（ESP32 + OLED + 蜂鸣器），**先不接红外**

**第 1 天**
- 上午：`Settings.h` 置 `ENABLE_SIM_MODE 1`，烧录；手机连热点开网页，
  逐个按钮验证进出 / 计费 / 蜂鸣 / OLED 全部联动 → **软件演示已可跑通**
- 下午：用 A4 / 卡纸搭停车场模型（第三节），摆小车、做招牌、装好 SG90 道闸闸杆

**第 2 天**
- 上午：按第四节脚本彩排 1~2 遍并录视频，多录一条备份
- 下午：（可选）简单剪辑 / 画中画；写一段提交说明；交付

**想"交的时候更完整一点"（有余力再做，别超范围）**
- 接 1~2 路真实红外（`ENABLE_SIM_MODE` 改回 0），补录一段"真车遮挡 → 自动入场"
  的真实镜头，与模拟镜头混剪，说明"模拟与真实传感器走同一套逻辑"
- 在视频里多讲计费规则（免费时长 + 按分钟向上取整 + 整数分运算）
- ⚠️ 不要临时加 RFID / 闸机 / 车牌识别 / 真实支付（见 [CLAUDE.md](../CLAUDE.md) 范围纪律），
  2 天内加这些只会翻车

## 六、常见问题

| 问题 | 处理 |
| --- | --- |
| 网页没有"演示控制"按钮 | 确认 `ENABLE_SIM_MODE` 已设为 1 并**重新烧录**；刷新网页 |
| 手机连不上热点 | 确认 `ENABLE_WEB=1`；热点名 `ESP32-Parking` / 密码 `parking123`（`Settings.h` 可改） |
| 现场来不及产生费用 | 调小 `PARKING_FREE_PERIOD_SEC`、调大 `PARKING_RATE_PER_MIN_CENTS`（`Settings.h`），录起来更快出数字 |
| OLED 不亮 | 改 `OLED_I2C_ADDRESS`（0x3C / 0x3D），检查 SDA/SCL 接线；不接也能只录网页 |
| 想纯录网页、连屏都不要 | 关 `ENABLE_DISPLAY`、`ENABLE_BUZZER` 仍可编译，只演示网页闭环 |
| 道闸方向反了（该抬杆却落杆） | 对调 `Settings.h` 的 `GATE_SERVO_OPEN_DEG` / `GATE_SERVO_CLOSED_DEG` |
| 舵机抖动 / 一动 ESP32 就重启 | 舵机改用独立 5V 供电并与 ESP32 共地；别用电脑 USB 弱口供电 |
| 不要道闸 | `Settings.h` 置 `ENABLE_GATE 0`，舵机代码退化为空实现，网页隐藏道闸 |
