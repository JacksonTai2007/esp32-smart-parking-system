# 硬件接线说明

引脚总表见 docs/pinout.md（代码权威定义：`config/Pins.h`）。
本文按模块给出具体接线、供电与排查要点。**接线前务必通读 pinout.md 的注意事项。**

> 本期方案：入口触摸感应做智能入场 + 自动分配车位引导，红外车位传感器做
> 车辆识别 + 按时长计费，火焰传感器做火灾报警联动风扇。
> **不做 RFID 刷卡、不用舵机闸机**；HC-SR04 入口超声波未纳入。
> 模块少、无大电流执行器，USB 5V 直接供电即可。

## 供电架构

```
USB 5V ──► ESP32 DevKit（板载稳压出 3.3V）
                │ 3.3V ──► OLED、红外车位模块(3.3~5V 均可)、触摸模块、火焰模块
                │ 5V(VIN) ──► 蜂鸣器模块（5V 供电音量更大）、继电器 + 风扇
                │
                └──► 所有模块 GND 必须与 ESP32 GND 连通（共地！）
```

- **所有地必须连通**：ESP32 GND 与各模块 GND 接在同一排

## 1. 红外车位传感器 × 2~4（车辆识别核心）

| 模块引脚 | 接到 |
| --- | --- |
| VCC | 3.3V（多数模块 3.3~5V 兼容） |
| GND | GND |
| OUT | 车位1→GPIO 32，车位2→GPIO 33，车位3→GPIO 34，车位4→GPIO 35 |

- 只接 2~3 个车位时，把 `Settings.h` 的 `TOTAL_PARKING_SLOTS` 改成实际数量，
  按顺序占用 32→33→34 引脚
- 常见红外避障模块**检测到物体输出 LOW**（板载 LED 亮）；
  若你的模块相反，把 `SLOT_OCCUPIED_LEVEL` 改为 `HIGH`
- 模块上的电位器可调检测距离，演示前调到稳定触发小车模型的距离
- 本系统把"车位被占用"当作"识别到一辆车"：占用瞬间开始计时，
  恢复空闲瞬间结算费用

## 2. OLED SSD1306（I2C）

| OLED 引脚 | 接到 |
| --- | --- |
| VCC | 3.3V |
| GND | GND |
| SDA | GPIO 21 |
| SCL | GPIO 22 |

不亮时排查：I2C 地址多数为 0x3C，少数 0x3D（改 `Settings.h` 的
`OLED_I2C_ADDRESS`）；可先跑 I2C scanner 示例确认。

## 3. 蜂鸣器

| 模块引脚 | 接到 |
| --- | --- |
| VCC | 3.3V |
| GND | GND |
| I/O | GPIO 14 |

默认按"高电平响"的**有源蜂鸣器**模块处理；若是低电平触发，
改 `Settings.h` 的 `BUZZER_ACTIVE_LEVEL = LOW`。无源蜂鸣器需 PWM 驱动，
本期不支持，购买时认准"有源/active buzzer"。

提示音节奏：入场 1 短响、离场（已计费）2 短响、满位 1 长响。

## 4. 入口触摸感应（TTP223，智能入场）

| 模块引脚 | 接到 |
| --- | --- |
| VCC | 3.3V |
| GND | GND |
| I/O（DO） | GPIO 25 |

- 拍一下感应片 = 识别"车辆到达入口"：自动分配空闲车位，网页横幅 + 车位高亮
  + OLED `>> Go to Px <<` 引导，蜂鸣 1 短响；满位则长响拒绝。
- TTP223 默认按下输出 HIGH（`ENTRY_ACTIVE_LEVEL`），引脚内部下拉，
  未接模块不会误触发；模拟模式下触摸同样有效。
- 不需要该功能时置 `ENABLE_ENTRY_GUIDE 0`，相关代码退化为空实现。

## 5. 火焰传感器 + 继电器风扇（火灾报警联动）

| 接线 | 接到 |
| --- | --- |
| 火焰模块 DO / VCC / GND | GPIO 27 / 3.3V / GND |
| 继电器 IN / VCC / GND | GPIO 16 / 5V / GND |
| 风扇（直流电机） | 继电器 COM→5V，NO→电机正极，电机负极→GND |

- 检测到火焰（确认 `FIRE_CONFIRM_MS`）→ 蜂鸣循环警报 + 风扇自动启动 +
  网页红色警报横幅；火焰消失 `FIRE_CLEAR_MS` 后自动解除。

## 5b. 雨滴 + 震动传感器（降雨提示 / 碰撞告警）

| 接线 | 接到 |
| --- | --- |
| 雨滴模块 DO / VCC / GND | GPIO 17 / 3.3V / GND |
| 震动模块 DO / VCC / GND | GPIO 18 / 3.3V / GND |

- 雨滴感应板沾水 → 网页黄色「检测到降雨」横幅，干燥后自动解除（不响铃）。
- 震动模块被敲击/晃动 → 蜂鸣 1 长响 + 网页橙色「疑似车辆碰撞」横幅，
  数秒自动消失；连续震动有冷却时间防刷屏。
- 两个模块都是比较器输出（默认触发 LOW），电位器可调灵敏度；触发电平在
  `Settings.h`（`RAIN_ACTIVE_LEVEL` / `IMPACT_ACTIVE_LEVEL`）。
- 不需要时置 `ENABLE_RAIN_ALERT 0` / `ENABLE_IMPACT_ALERT 0`。
- 课堂演示：红外遥控器对准火焰传感器按键即可安全触发，无需明火。
- 继电器为低电平吸合时把 `FAN_ACTIVE_LEVEL` 改 `LOW`；
  不需要该功能时置 `ENABLE_FIRE_ALARM 0`。

## 上电自检顺序（首次接线后）

1. 只接 ESP32 + OLED → 烧录 → 看 OLED 出画面、串口出 banner
2. 加蜂鸣器 → 遮挡/移开车位传感器应分别听到入场 1 短响、离场 2 短响
3. 加红外车位 → 逐个遮挡看车位日志（占用/空闲翻转、剩余车位变化）
4. 加触摸模块 → 拍一下感应片，串口看 `[PM] entry: assigned P1`，网页出现
   引导横幅、OLED 显示 `>> Go to P1 <<`
5. 加火焰 + 继电器风扇 → 遥控器对准火焰传感器按一下：循环警报 + 风扇转 +
   红色横幅；移开约 2 秒自动解除
6. 完整流程：拍触摸 → 分配 P1 → 遮挡 P1 红外（入场计时）→ 等待几秒 → 移开 →
   看串口/OLED/网页显示本次停留时长与费用、累计收入累加

完整测试清单见 tests/README.md。
