# 实物硬件清单（套件：ESP32 高配版）

本文件是**实物套件里"有哪些模块"的权威记录**。它与 README §4 的分工是：

- 本表 = **手上实际拥有的全部硬件**（高配版，33 项 + 一块普中一体化实验板）
- [README §4 硬件清单](../README.md#4-硬件清单) = **本停车场系统当前真正用到的子集**
- [docs/pinout.md](pinout.md) / [`config/Pins.h`](../firmware/parking-system/config/Pins.h)
  = 已接线模块的**引脚权威定义**

> ⚠️ 本套件是**高配版**（厂家分 低配基础版 / 中配版 / 高配版 三档）。
> 规划与选型时**不要按低配版或中配版假设缺少模块**——下表中标为"备用/候选"
> 的模块都是实物已到、随时可用的，只是当前 Phase 1 MVP 没有接入。

## 开发板说明

主控为**普中-ESP32 一体化实验板**（核心板为 ESP32-WROOM-32，可插拔）。
载板上集成了若干板载外设，照片可见：4 路独立按键、RGB LED、4 位数码管、
有源蜂鸣器、继电器（SONGLE SRD-05VDC）、直流电机驱动接口、红外接收头、
ADC 电位器，以及 OLED/LCD、传感器模块的排针接口。

- 本项目目前按**独立模块 + 杜邦线**的方式接线，引脚以 [`config/Pins.h`](../firmware/parking-system/config/Pins.h) 为准。
- 载板**板载外设的固定 GPIO 映射以板上丝印 / 厂家手册为准**；云端无法核对实物，
  若后续想直接复用板载蜂鸣器、继电器、数码管等，需本地对照丝印确认引脚，
  可能与现有 `Pins.h` 不同，**以实际接线为准**。

## Phase 2 候选用途（仅建议，尚未实现）

下列只是"实物已具备、可在后续阶段考虑"的方向，**当前不接线、不写代码**，
列出来是为了答辩/论文展望章节有据可依：

1. **消防联动（已规划）**：火焰传感器(#14) + 烟雾传感器(#16) 触发蜂鸣器报警，
   直流电机/风扇(#2) 排风——引脚已在 `Pins.h` 预留。
2. **环境数据展示**：DHT11(#26) 或 DS18B20(#6) 在 OLED/Web 上多一页温湿度。
3. **入口/出口过车计数**：激光(#33) + U 型光电(#32) 组成对射光幕，得到比
   超声波更干脆的"有车通过"边沿信号，可做进出场统计与计费基础。
4. **闸机选型对比**：28BYJ-48 步进电机(#21) 作为 SG90 之外的执行机构对比方案。
5. **本地交互增强**：红外遥控器(#3) / 触摸开关(#9) / 旋转编码器(#29) 做手动
   开闸、复位、参数调节，减少对 Web 的依赖。

> 📌 **范围纪律**（见 [CLAUDE.md](../CLAUDE.md)）：本期**不**实现消防报警逻辑、
> **不**一次性接入所有高级传感器、**不**做车牌识别/真实支付/云数据库。
> 任何新模块接入都要先在 `config/Pins.h` 分配引脚、同步 `docs/pinout.md` 与 README，
> 并检查不与现有引脚、ESP32 启动敏感引脚 / 输入专用脚冲突。
