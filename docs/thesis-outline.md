# 毕业论文大纲（建议稿）

题目：基于 ESP32 的智能停车场管理系统设计与实现

> 本大纲与仓库实现对应：写作时第 3、4 章直接引用 docs/pinout.md、
> docs/architecture.md 与源码，第 5 章引用 tests/README.md 的实测记录。

## 第 1 章 绪论
1.1 研究背景与意义（停车难、车位利用率、物联网停车管理趋势）
1.2 国内外研究现状（传统出入口管理、车位检测与诱导、ETC/车牌识别、云平台方案对比）
1.3 本文主要工作与论文结构
1.4 开发方法说明：云端 AI Agent 协作开发流程（特色点，可写一节：
    Claude Code Cloud 主开发 + Codex Cloud 审查 + GitHub CI + 本地硬件验证）

## 第 2 章 系统总体设计
2.1 需求分析（功能性：车位识别、车位管理、按时长计费与展示；非功能性：实时性、可扩展）
2.2 总体方案与系统框图（感知层—控制层—展示层；以红外车位识别驱动全流程）
2.3 主控选型：ESP32（双核、Wi-Fi、外设资源；与 Arduino UNO/STM32 对比）
2.4 传感器与提示器件选型（红外车位传感器、SSD1306 OLED、蜂鸣器；
    说明本期不引入 RFID/超声波/舵机的取舍理由）

## 第 3 章 系统硬件设计
3.1 硬件总体结构与供电设计（USB 5V / 各传感器共地）
3.2 各模块接口电路
    3.2.1 车位检测电路（红外避障，输入专用脚 GPIO 34/35 说明）
    3.2.2 显示电路（I2C OLED SSD1306，地址 0x3C/0x3D）
    3.2.3 提示电路（蜂鸣器，有源/电平触发说明）
3.3 引脚分配与启动敏感引脚规避（引用 docs/pinout.md；含 Phase 2 火焰/烟雾/风扇预留脚）

## 第 4 章 系统软件设计
4.1 软件总体架构（模块划分、单一职责，引用 docs/architecture.md）
4.2 车位事件驱动的业务流程设计（车位"空闲↔占用"翻转 → 入场计时 / 离场计费 / 满位提示）
4.3 计费算法设计（按停留时长、整数「分」运算避免浮点误差、按分钟向上取整、
    免费时长抗短暂停靠误触发）
4.4 停车会话与营收统计（单次会话记录、累计收入与停车次数、最近记录环形缓冲）
4.5 非阻塞调度设计（millis() 方案、与 delay() 方案对比、溢出安全）
4.6 关键模块实现
    4.6.1 红外车位检测与软件去抖
    4.6.2 计费与营收统计（BillingService 纯逻辑）
    4.6.3 状态快照与 OLED/Web 只读消费
4.7 Web 监控设计（HTTP 服务、JSON 状态 API、清零收入接口、AP 兜底模式）
4.8 软件工程实践（配置集中化、条件编译、CI 自动编译验证）

## 第 5 章 系统测试
5.1 测试环境与方法（编译测试 / 模块测试 / 整机联调，引用 tests/README.md）
5.2 功能测试（车位识别入场、离场计费、满位处理、Web 查看与清零——附串口日志与照片）
5.3 计费正确性测试（免费时长内为 0、超出后时长×单价、向上取整、累计与最近记录）
5.4 参数标定（红外灵敏度、触发电平、去抖时间、计费单价与免费时长）
5.5 问题与解决（记录实测中踩的坑：电平逻辑、去抖、供电、I2C 地址等）
