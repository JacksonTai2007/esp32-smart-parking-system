# 毕业论文大纲（建议稿）

题目：基于 ESP32 的智能停车场管理系统设计与实现

> 本大纲与仓库实现对应：写作时第 3、4 章直接引用 docs/pinout.md、
> docs/architecture.md 与源码，第 5 章引用 tests/README.md 的实测记录。

## 第 1 章 绪论
1.1 研究背景与意义（停车难、车位利用率、物联网停车管理趋势）
1.2 国内外研究现状（传统刷卡闸机、ETC/车牌识别、云平台方案对比）
1.3 本文主要工作与论文结构
1.4 开发方法说明：云端 AI Agent 协作开发流程（特色点，可写一节：
    Claude Code Cloud 主开发 + Codex Cloud 审查 + GitHub CI + 本地硬件验证）

## 第 2 章 系统总体设计
2.1 需求分析（功能性：入场控制、车位监测、状态展示；非功能性：实时性、可扩展）
2.2 总体方案与系统框图（感知层—控制层—展示层）
2.3 主控选型：ESP32（双核、Wi-Fi、外设资源；与 Arduino UNO/STM32 对比）
2.4 传感器与执行器选型（RC522、HC-SR04、红外、SG90、SSD1306、蜂鸣器）

## 第 3 章 系统硬件设计
3.1 硬件总体结构与供电设计（USB 5V / 舵机独立 5V / 共地）
3.2 各模块接口电路

    3.2.2 闸机舵机驱动（PWM、独立供电）
    3.2.3 入口检测电路（HC-SR04，Echo 分压设计）
    3.2.4 车位检测电路（红外，输入专用脚说明）
    3.2.5 显示与提示电路（I2C OLED、蜂鸣器）
3.3 引脚分配与启动敏感引脚规避（引用 docs/pinout.md）

## 第 4 章 系统软件设计
4.1 软件总体架构（模块划分、单一职责，引用 docs/architecture.md）
4.2 主业务状态机设计（状态图、各状态迁移条件）
4.3 非阻塞调度设计（millis() 方案、与 delay() 方案对比、溢出安全）
4.4 关键模块实现

    4.4.2 闸机控制（无反馈舵机的计时状态推进）

4.5 Web 监控设计（HTTP 服务、JSON API、AP 兜底模式）
4.6 软件工程实践（配置集中化、条件编译、CI 自动编译验证）
